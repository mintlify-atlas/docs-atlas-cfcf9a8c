# Quick start

Get started with Desloppify by scanning your first codebase. This guide will walk you through running a scan, viewing results, and fixing your first issue.

## Prerequisites

Make sure you have [installed Desloppify](/installation) before continuing.

## Your first scan

<Steps>
  <Step title="Navigate to your project">
    Open your terminal and navigate to the project you want to scan:

    ```bash
    cd /path/to/your/project
    ```
  </Step>

  <Step title="Run your first scan">
    Scan your entire project or a specific directory:

    <CodeGroup>
      ```bash Scan entire project
      desloppify scan --path .
      ```

      ```bash Scan specific directory
      desloppify scan --path src/
      ```
    </CodeGroup>

    The scan will:
    - Auto-detect your project's language(s)
    - Run all relevant detectors
    - Calculate quality scores
    - Generate a scorecard badge

    <Info>
      First scans may take a few minutes depending on codebase size. Subsequent scans are incremental and faster.
    </Info>
  </Step>

  <Step title="View your results">
    After the scan completes, check your quality score and progress:

    ```bash
    desloppify status
    ```

    You'll see:
    - Overall quality score
    - Objective and strict scores
    - Per-tier progress breakdown
    - Dimension scores (if available)

    Example output:
    ```
    Overall: 87.3  Objective: 89.1  Strict: 85.7

    Tier breakdown:
    T1 (auto-fix):     12 open
    T2 (quick):        8 open
    T3 (judgment):     5 open
    T4 (refactor):     2 open
    ```
  </Step>

  <Step title="Get your next task">
    Desloppify prioritizes issues for you. Get the highest-priority item to fix:

    ```bash
    desloppify next
    ```

    This shows:
    - The issue to fix
    - Which file it's in
    - Why it matters
    - The resolve command to run when done

    Example output:
    ```
    [T2] Unused export in src/utils/helpers.ts
    Function 'formatDate' is exported but never imported

    Suggestion: Remove the export or delete if truly unused

    When fixed:
    desloppify resolve fixed helpers.ts
    ```
  </Step>

  <Step title="Fix the issue">
    Open the file and fix the issue. For this example:

    ```typescript src/utils/helpers.ts
    // Before: exported but unused
    export function formatDate(date: Date): string {
      return date.toISOString();
    }

    // After: removed unused export
    function formatDate(date: Date): string {
      return date.toISOString();
    }
    ```
  </Step>

  <Step title="Mark it resolved">
    After fixing the issue, mark it as resolved:

    ```bash
    desloppify resolve fixed helpers.ts
    ```

    You can also use patterns or IDs:

    <CodeGroup>
      ```bash By file
      desloppify resolve fixed src/utils/helpers.ts
      ```

      ```bash By directory
      desloppify resolve fixed src/utils/
      ```

      ```bash By detector
      desloppify resolve fixed unused
      ```
    </CodeGroup>
  </Step>

  <Step title="Continue improving">
    Run `next` again to get the next priority item:

    ```bash
    desloppify next
    ```

    Keep this cycle going:
    1. Run `next` to get the top priority issue
    2. Fix the issue
    3. Mark it resolved
    4. Repeat
  </Step>
</Steps>

## Auto-fix mechanical issues

Some issues (T1) can be auto-fixed:

```bash
desloppify fix unused --dry-run
```

Remove `--dry-run` to apply the fixes:

```bash
desloppify fix unused
```

<Warning>
  Always review auto-fixes before committing. Use `--dry-run` first to see what will change.
</Warning>

## Check specific areas

You can scope your view to specific files or directories:

<CodeGroup>
  ```bash Show issues in a file
  desloppify show src/components/Header.tsx
  ```

  ```bash Show issues in a directory
  desloppify show src/components/
  ```

  ```bash Show issues by detector
  desloppify show unused
  ```
</CodeGroup>

## Rescan to refresh

After fixing several issues, rescan to update your scores:

```bash
desloppify scan --path .
```

The scan is incremental and will update findings based on your changes.

## Configuration tips

### Set a target score

Set your quality score goal:

```bash
desloppify config set target_strict_score 95
```

### Exclude paths

Exclude files or directories from scanning:

```bash
desloppify scan --path . --exclude migrations --exclude tests
```

Or persist exclusions in config:

```bash
desloppify config set exclude '["migrations", "tests", "node_modules"]'
```

### Custom badge path

Change where the scorecard badge is saved:

```bash
desloppify config set badge_path assets/quality-badge.png
```

## Common workflows

### For AI agents

If you're an AI agent working to improve codebase quality:

```bash
# Install and setup
pip install --upgrade "desloppify[full]"
desloppify update-skill claude  # or your agent type

# Initial scan
desloppify scan --path .

# Fix loop
desloppify next      # get priority issue
# ...fix the issue...
desloppify resolve fixed <pattern>
desloppify next      # repeat
```

### For human developers

Quick quality check before committing:

```bash
# Scan changed areas
desloppify scan --path src/

# Check status
desloppify status

# Fix auto-fixable issues
desloppify fix unused --dry-run
desloppify fix unused

# Review top priorities
desloppify next --count 5
```
