## Overview

The `next` command shows the highest-priority items from your work queue. It uses intelligent ranking that considers tier, dimension impact, plan overrides, and cluster focus.

## Basic Usage

```bash
desloppify next
```

Shows the single highest-priority item to work on.

## Command Options

### Queue Configuration

<ParamField path="--count" type="integer">
  Number of items to show. Default: `1`
  
  Example: `--count 10` shows top 10 items
</ParamField>

<ParamField path="--tier" type="integer">
  Show only items from a specific tier. Choices: `1`, `2`, `3`, `4`
  
  - T1 = Critical (security, data loss)
  - T2 = Important (bugs, correctness)
  - T3 = Moderate (quality, duplication)
  - T4 = Cosmetic (style, subjective)
</ParamField>

<ParamField path="--scope" type="string">
  Optional scope filter. Can be:
  - File path: `src/components/Button.tsx`
  - Directory: `src/auth/`
  - Detector name: `unused`
  - ID prefix: `security::src/`
  - Glob pattern: `src/**/*.ts`
</ParamField>

<ParamField path="--status" type="string">
  Status filter for queue items. Default: `open`
  
  Choices: `open`, `fixed`, `wontfix`, `false_positive`, `auto_resolved`, `all`
</ParamField>

### Display Options

<ParamField path="--group" type="string">
  Group output by category. Default: `item`
  
  Choices:
  - `item` - Show individual findings
  - `file` - Group by file
  - `detector` - Group by detector type
  - `tier` - Group by priority tier
</ParamField>

<ParamField path="--format" type="string">
  Output format. Default: `terminal`
  
  Choices:
  - `terminal` - Colorized terminal output
  - `json` - JSON data structure
  - `md` - Markdown format
</ParamField>

<ParamField path="--output" type="string">
  Write output to file instead of stdout. Use with `--format json` or `--format md`.
  
  Example: `--output queue.json --format json`
</ParamField>

### Advanced Filtering

<ParamField path="--cluster" type="string">
  Filter to a specific plan cluster. If you have an active cluster set via `plan focus`, it auto-applies.
  
  Example: `--cluster auth-refactor`
</ParamField>

<ParamField path="--include-skipped" type="boolean">
  Include items marked as skipped (temporary, permanent, or false positive) in the queue.
</ParamField>

<ParamField path="--no-tier-fallback" type="boolean">
  Do not auto-fallback to another tier when `--tier` has no items. By default, if your specified tier is empty, Desloppify shows the next populated tier.
</ParamField>

### Debugging

<ParamField path="--explain" type="boolean">
  Show ranking and tier-fallback rationale. Useful for understanding why items are ordered the way they are.
</ParamField>

### Global Options

<ParamField path="--state" type="string">
  Path to state file. Default: `.desloppify/state.json`
</ParamField>

<ParamField path="--lang" type="string">
  Language context for narrative generation.
</ParamField>

## Tier System

Findings are prioritized by tier:

| Tier | Priority | Description | Examples |
|------|----------|-------------|----------|
| T1 | Critical | Security issues, data loss risks | SQL injection, null pointer dereference |
| T2 | Important | Bugs, correctness, major smells | Logic errors, API misuse |
| T3 | Moderate | Code quality, duplication, dead code | Unused imports, duplicate code |
| T4 | Cosmetic | Style, subjective review dimensions | Naming, structure, API design |

## Examples

### Show single next item

```bash
desloppify next
```

### Show top 10 items

```bash
desloppify next --count 10
```

### Show only critical (T1) items

```bash
desloppify next --tier 1
```

### Show next item in specific directory

```bash
desloppify next --scope src/auth/
```

### Show next items grouped by file

```bash
desloppify next --count 20 --group file
```

### Show with explanation of ranking

```bash
desloppify next --explain
```

### Show items in a cluster

```bash
desloppify next --cluster auth-refactor --count 5
```

### Export queue to JSON

```bash
desloppify next --count 50 --format json --output queue.json
```

### Show all statuses (including resolved)

```bash
desloppify next --status all --count 20
```

## Integration with Living Plan

The `next` command respects your living plan:

- **Custom ordering** - Items moved via `plan move` appear first
- **Cluster focus** - When a cluster is active, only show items in that cluster
- **Skip tracking** - Skipped items are hidden (unless `--include-skipped`)
- **Done tracking** - Completed items are excluded
