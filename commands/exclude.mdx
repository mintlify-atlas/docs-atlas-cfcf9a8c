## Overview

The `exclude` command adds glob patterns to your project configuration, marking files and directories to skip during scanning. This is useful for ignoring generated code, vendor dependencies, build artifacts, and other files you don't want analyzed.

## Usage

```bash
desloppify exclude <pattern>
```

## Arguments

| Argument | Description |
|----------|-------------|
| `pattern` | Glob pattern matching files or directories to exclude |

## Examples

```bash
# Exclude all Python cache files
desloppify exclude "__pycache__"

# Exclude node_modules directory
desloppify exclude "node_modules"

# Exclude all compiled Python files
desloppify exclude "*.pyc"

# Exclude build directory and contents
desloppify exclude "build/**"

# Exclude multiple patterns (run command multiple times)
desloppify exclude "*.min.js"
desloppify exclude "dist/*"
desloppify exclude ".venv"
```

## Glob pattern syntax

<AccordionGroup>
  <Accordion title="Wildcards">
    - `*` - Matches any characters except `/`
    - `**` - Matches any characters including `/` (recursive)
    - `?` - Matches any single character
    - `[abc]` - Matches any character in the set
  </Accordion>

  <Accordion title="Examples">
    - `*.pyc` - All files ending in `.pyc`
    - `test_*.py` - All files starting with `test_`
    - `**/__pycache__/**` - All `__pycache__` directories anywhere
    - `build/**/*.js` - All `.js` files under `build/`
    - `*.{min.js,map}` - Files ending in `.min.js` or `.map`
  </Accordion>

  <Accordion title="Relative vs absolute">
    Patterns are matched relative to the project root. Use `**/` prefix to match at any depth:
    - `src/temp` - Only matches `src/temp` at root
    - `**/temp` - Matches any `temp` directory anywhere
  </Accordion>
</AccordionGroup>

## Behavior

<Info>
After adding an exclude pattern, Desloppify automatically sets `needs_rescan = true` in the configuration. Run `desloppify scan` to apply the exclusion.
</Info>

**What happens:**
1. Pattern is added to `exclude_patterns` list in `.desloppify/config.json`
2. Configuration is saved with `needs_rescan` flag
3. A warning is displayed if a rescan is needed
4. Next scan will skip matching files

**Idempotency:**
- Adding the same pattern multiple times is safe
- Patterns are stored in a list (duplicates are kept but have no extra effect)

## Common exclusion patterns

### Python projects
```bash
desloppify exclude "__pycache__"
desloppify exclude "*.pyc"
desloppify exclude ".pytest_cache"
desloppify exclude ".tox"
desloppify exclude ".venv"
desloppify exclude "*.egg-info"
```

### JavaScript/TypeScript projects
```bash
desloppify exclude "node_modules"
desloppify exclude "dist"
desloppify exclude "build"
desloppify exclude "*.min.js"
desloppify exclude "coverage"
```

### General build artifacts
```bash
desloppify exclude "target"
desloppify exclude "out"
desloppify exclude ".git"
desloppify exclude ".DS_Store"
```

## Viewing current exclusions

To see all active exclude patterns:

```bash
desloppify config
```

Look for the `exclude_patterns` entry in the output.

## Removing exclusions

There's no dedicated `unexclude` command. To remove patterns:

1. Edit `.desloppify/config.json` directly
2. Remove unwanted patterns from the `exclude_patterns` array
3. Run `desloppify scan` to re-analyze previously excluded files

Alternatively, reset to defaults:

```bash
desloppify config unset exclude_patterns
```

## When to use

<AccordionGroup>
  <Accordion title="Generated code">
    Exclude code generated by tools (protobuf, GraphQL, ORM migrations) that you don't manually edit:
    ```bash
    desloppify exclude "src/generated/**"
    desloppify exclude "*_pb2.py"
    ```
  </Accordion>

  <Accordion title="Third-party dependencies">
    Skip vendored libraries and package managers:
    ```bash
    desloppify exclude "node_modules"
    desloppify exclude "vendor"
    ```
  </Accordion>

  <Accordion title="Build artifacts">
    Ignore compiled output and temporary files:
    ```bash
    desloppify exclude "dist"
    desloppify exclude "*.pyc"
    desloppify exclude ".cache"
    ```
  </Accordion>

  <Accordion title="Performance optimization">
    Large codebases scan faster when irrelevant files are excluded. Focus analysis on source code you maintain.
  </Accordion>
</AccordionGroup>
